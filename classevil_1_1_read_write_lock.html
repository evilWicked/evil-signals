Detailed Description

a Read Write implementation to allow parallel reads and sequential writes.

The ReadWriteLock enables multiple reader to exist concurrently while restricting writes to to sequential access. The particular scenario is a list that has readers progressing down it at those times we are not making changes to the structure of the list.

The CReadWriteLock has the following behaviours. 

1. Reads can occur in parallel.
2. Writes can only occur when no other read or write is in progress.
3. Reads can not occur when a write is in progress.
