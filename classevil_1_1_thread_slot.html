Detailed Description

template<typename... Args>
class evil::ThreadSlot< Args >

The thread safe specialisation of the underlying SlotBase class.

The ThreadSlot class is intended to be used in conjunction with the ThreadSignal class. This version provides a threadsafe specialisation of the underlying signals and slots mechanism. What this guarantees is that you can interact with both the signal and slot from mutilple threads simultaneously without conflict.

It does not solve all the problems related to threading.

In particular if you grab a pointer to associated signal from the slot using slot.signal() and wish to use it you will still need to ensure that no other thread has deleted the slot while you are holding it.
